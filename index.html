HTML

  • Hypertext Markup Language (HTML)
    • Text elements
    • Image elements
    • Header elements
    • List elements
    • Meta-related elements
    • Content division elements
    • Semantic elements
  • HTML elements have an opening tag and a closing tag represented with angle brackets.
  • The head element contains information about the webpage that helps to render the page properly. These elements are are not visible when a user looks at a webpage
  • The style element defines the CSS styles associated with a website.
  • The link element connects an external resource to the HTML document.
  • The body element represents the visible content shown to the user.
  • The meta element defines metadata such as the character set (UTF-8), description, keywords, author, and viewport.

CSS

  • Cascading Style Sheets
  • The most common way to style a webpage is by using an external CSS style sheet. This is preferred in most cases because it allows developers to keep all their CSS rules in a separate file, which makes design changes easier.
  • When using an external style sheet, link the file using a link element placed in the HTML's head.
  • Separation of concerns refers to separating code into different files based on use
  • Declarations contain two important components: the CSS property we want to apply and the value of the property. For example, the declaration color: blue; includes the CSS property color and the value for that property, blue.
  • Instead of repeating a rule which would add more lines of code and repetition, two or more elements that will share the same styling can be grouped together. This helps us write Do Not Repeat Yourself (DRY) code.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • Git is a version control system that lets you manage and keep track of each version of your project.
  • Git commands always start with "Git"
  • The default branch is the "main" branch but "master" may also be seen in older repos. We should not do any coding in the main branch. Branch off and create a "feature" branch instead to maintain the main branch.
  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -a: adds modifications in the current working branch to the staging area and "-a" shows that we want to add all changes.
  • git commit -m "commit message describing what the commit contains"
  • git pull origin main: receives the main branch modifications into the local environment from the cloud repo.
  • git push origin feature: pushes the modifications on a feature branch onto the cloud repo.
  • The first step to creating a local development version is to create a feature branch.
  • A pull request, or PR, is used to merge a comparison branch into the base branch.

Javascript

  • A variable is a named container that allows us to store data in our code. To declare a variable, we use the var keyword. After the var keyword, we give the variable a name. We use an assignment operator = to assign the specific data that we want stored in our variable.
  • All strings are surrounded by quotation marks.
  • Each line of JavaScript code ends with a semicolon.
  • Use the console log in ChromeDevTools to check your code.
  • Control flow is the order in which a computer executes code in a script. JavaScript reads top to bottom.
  • Example statment using else if : if (first condition) { // block of code to be executed if first condition is truthy } else if (second condition) { // block of code to be executed if second condition is truthy }
  • Example of if statment: if (0) { console.log("I'm 0!") } else if (null) { console.log("I'm null!") } else if (true) { console.log("I'm true!") }
  • An array is a single variable that is used to hold a group of data.
  • A for loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
  • Commenting out code makes it inactive without deleting it, just in case you need it later. To comment out multiple lines of JavaScript code, put /* before and */ after the block of code
  • A function is simply a set of instructions that tells the computer how to perform a certain task.
  • Math.floor() and Math.random() are both properties of JavaScript's built-in Math object that can be used together to generate a random number.
